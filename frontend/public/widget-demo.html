Quick Start

Add this code to your website, right before the closing </body> tag:

HTML 
<!-- AgentForge Widget -->
<script src="https://your-domain.com/agentforge-widget.js"></script>
<script>
  AgentForge.init({
    apiKey: 'af_xxxxxxxxxxxxx',
    apiUrl: 'https://your-api.com/api/v1/public',
    position: 'bottom-right',
    primaryColor: '#3B82F6',
    title: 'Chat with us!',
    subtitle: 'We typically reply in a few minutes'
  });
</script>
Important: Replace af_xxxxxxxxxxxxx with your actual API key. You can generate API keys from your agent's dashboard in the AgentForge platform.

Configuration Options

Customize the widget to match your needs:

Parameter Type Status Description
apiKey string Required Your AgentForge API key
apiUrl string Required API endpoint URL
position string Optional 'bottom-right' or 'bottom-left' (default: 'bottom-right')
primaryColor string Optional Main color in hex format (default: '#3B82F6')
title string Optional Widget title (default: 'Chat with us!')
subtitle string Optional Widget subtitle (default: 'We typically reply in a few minutes')
placeholder string Optional Input placeholder (default: 'Type your message...')
avatar string Optional Avatar image URL
welcomeMessage string Optional First message displayed to users (default: 'Hi! 👋 How can I help you today?')
zIndex number Optional CSS z-index (default: 9999)

How to Get Your API Key

  1. Navigate to your AgentForge dashboard at your deployment URL
  2. Select the agent you want to embed on your website
  3. Click on the "API Keys" tab in the agent's interface
  4. Click the "New API Key" button to generate a new key
  5. Give your key a descriptive name (e.g., "Production Website")
  6. Optionally configure allowed origins and rate limits
  7. Copy the generated API key (starts with af_) - you won't be able to see it again
  8. Paste the key into your widget configuration code

Security Best Practices

  • Only use API keys on domains you own and trust
  • Configure allowed origins in your API key settings to restrict usage to specific domains
  • Monitor your API usage regularly in the AgentForge dashboard
  • Set appropriate rate limits to prevent abuse
  • Rotate API keys periodically (e.g., every 90 days)
  • Immediately disable or delete API keys if compromised
  • Never commit API keys to public repositories
  • Use environment variables for API keys in your build process
